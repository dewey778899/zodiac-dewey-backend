## 代码审查 & Bug 修复 (19:39~)

### 发现并修复的 Bug

1. **`RateLimitService.tryAcquireWechat` 竞态条件**（高优先级）
   - 问题：`count.get()` + `count.incrementAndGet()` 非原子操作，高并发时可能超限
   - 修复：改用 CAS 循环（`compareAndSet`），保证原子性
   - 文件：`RateLimitService.java`

2. **`PayService.SecureRandom` 重复实例化**（性能）
   - 问题：`generateToken()` 和 `generateRandom()` 每次调用都 `new SecureRandom()`，开销大
   - 修复：提取为类级别静态字段 `private static final SecureRandom RNG = new SecureRandom()`
   - 文件：`PayService.java`

3. **`queryStatus` 中 `transaction_id` 空字符串覆盖**（数据一致性）
   - 问题：PayJS check 返回 status=1 但 transaction_id 为空时，仍会 save 覆盖回调已写入的值
   - 修复：仅在 `txId` 非空时才 `setTransactionId(txId)`
   - 文件：`PayService.java`

4. **订单描述与实际模型名不一致**（文案）
   - 问题：`body` 写成 "Claude Opus 4.7"，但实际配置是 `claude-sonnet-4-6-20260217`
   - 修复：改为 "星座深度合盘解析 · Claude 深度版"，与实际模型解耦
   - 文件：`PayService.java`

### 发现的系统级注意事项
- WorkBuddy 会向系统注入 `SERVER__PORT=62284`（双下划线格式），被 Spring Boot Relaxed Binding 识别为 `server.port`，导致后端试图绑定 62284 端口失败
- 解决：启动时加 `--server.port=8080` 命令行参数（优先级高于 OS 环境变量）
